## 📚 Parte 1: Introduzione a Jupyter Notebook

### Cosa Ho Scoperto Essere Jupyter

**Jupyter Notebook** = ambiente di programmazione **interattivo** che combina:
- Codice eseguibile (celle Python)
- Testo formattato (Markdown)
- Output (grafici, tabelle, risultati)
- Tutto salvato in un singolo file `.ipynb` (JSON)

**Differenza chiave vs script tradizionale `.py`:**
- Script: esecuzione lineare dall'alto in basso, ogni volta da zero
- Notebook: esecuzione celle **a scelta**, stato **persistente**, iterazione rapida

### Quando Usare Jupyter

✅ **Sì:** Esplorazione dati, apprendimento, prototipazione, visualizzazioni  
❌ **No:** Codice production, script automatizzati, applicazioni

**Caso mio:** Ideale per Think Python perché molti esempi sono interattivi (turtle graphics, esperimenti con funzioni).

## 🏗️ Parte 2: Architettura Jupyter (Come Funziona)

### Componenti Principali

```
Browser (Interface) ←→ Jupyter Server (Backend) ←→ Kernel (Python Engine)
```

1. **Browser/Interface:** Dove scrivi codice e vedi output
2. **Jupyter Server:** Processo HTTP che gestisce richieste, salva file, coordina
3. **Kernel:** Interprete Python separato che **esegue effettivamente** il codice

**Concetto chiave:** Il kernel mantiene lo **stato** (variabili in memoria) tra esecuzioni celle.

### Esempio Comportamento Kernel

```python
# Cella 1
x = 10

# Cella 2 (eseguita dopo)
print(x)  # ✓ Funziona, x esiste ancora!

# Cella 3 (se eseguita PRIMA della Cella 2)
x = 100

# Ri-esegui Cella 2
print(x)  # Output: 100 (non 10!)
```

**Lezione:** L'ordine di esecuzione conta, non l'ordine nel file!

### Formato `.ipynb`

File JSON strutturato così:
```json
{
  "cells": [
    {"cell_type": "markdown", "source": ["# Titolo"]},
    {"cell_type": "code", "source": ["x = 42"], "outputs": [...]}
  ],
  "metadata": {...}
}
```

**Implicazioni:**
- ✅ Versionabile con Git (è testo)
- ✅ Funziona ovunque (standard aperto)
- ⚠️ Git diff poco leggibile (serve nbdime)

## 🔧 Parte 3: Setup Jupyter - Due Modalità

### Modalità 1: Jupyter Classico (Server Separato)

**Comando:**
```powershell
jupyter notebook
```

**Cosa succede:**
1. Processo server si avvia (`http://localhost:8888`)
2. Browser apre automaticamente
3. Terminale rimane bloccato (Ctrl+C per fermare)

**Pro/Contro:**
- ✅ Interfaccia "pura" Jupyter
- ✅ Accessibile da altri dispositivi (via rete)
- ❌ Terminale occupato
- ❌ Devi ricordare di fermare server

### Modalità 2: VS Code Integration (Nessun Server)

**Setup:**
1. Installa estensione "Jupyter" in VS Code
2. Crea file `.ipynb`
3. Tutto funziona dentro VS Code

**Differenza chiave:** VS Code **elimina il server HTTP** e gestisce tutto internamente.

**Pro/Contro:**
- ✅ Nessun terminale occupato
- ✅ Editor familiare
- ✅ Git integration migliore
- ❌ Non è l'interfaccia "classica" Jupyter

**Il kernel Python è identico in entrambe le modalità!**

## 🤔 Parte 5: Virtual Environments - La Discussione

### Domanda Chiave

"Ho usato conda/venv in passato. È possibile che pandas/numpy non siano globali?"

**Risposta:** SÌ! Gli ambienti virtuali sono **isolati**.

```
Sistema Windows
├─ Python Globale → jupyter, setuptools
├─ Ambiente Conda "data_science" → numpy, pandas, matplotlib
└─ Ambiente venv "altro_progetto" → django, requests
```

Ogni ambiente ha **propri pacchetti**, invisibili agli altri.

### Mia Scelta Attuale

**Ambiente globale (no venv)** per ora.

**Motivazione:**
- Prima imparo Python
- Poi imparo gestione ambienti
- Accetto rischio "inquinamento"

**Trade-offs accettati:**
- ✅ Semplicità: zero configurazione
- ✅ Focalizzazione: studio Python, non DevOps
- ❌ Conflitti potenziali: monitorati ma accettati
- ❌ Riproducibilità limitata: documentata via snapshot

**Piano futuro:** Migrazione a conda/venv in Fase 2 (dopo capitoli PCC 1-11).

## 🎓 Parte 7: Concetti Chiave Appresi

### 1. Jupyter != Solo Editor

È un **ecosistema** completo:
- Server HTTP (backend)
- Kernel Python (esecuzione)
- Interface (browser o VS Code)
- Formato file standardizzato (.ipynb)

### 2. Kernel Mantiene Stato

Non come script che parte da zero ogni volta:
```python
# Esegui questa cella 3 volte
x = x + 1 if 'x' in dir() else 1
print(x)
# Output: 1, 2, 3 (non sempre 1!)
```

**Soluzione:** "Restart Kernel" quando serve ricominciare.

### 3. Virtual Environments Sono Isolati

`pip list` mostra solo pacchetti dell'ambiente **attivo**.

Se ho pacchetti in ambiente conda, devo:
```powershell
conda activate nome_ambiente
pip list  # Ora vedo pacchetti conda
```

### 4. `.ipynb` È JSON

Quindi:
- ✅ Versionabile Git
- ❌ Diff poco leggibile (soluzione: nbdime)
- ⚠️ Output salvati nel file (aumenta dimensione)

**Best practice:** Clear output prima di commit.

### 5. Jupyter Ha Due Interfacce

- **Notebook** (classica): singolo notebook, semplice
- **JupyterLab** (moderna): multi-tab, IDE-like, più potente

Posso usare entrambe, stesso kernel sotto.
